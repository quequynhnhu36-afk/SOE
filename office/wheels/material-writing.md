# Material Writing Wheel

## 适用场景

用于领导讲话、工作汇报、年度或阶段总结、经验交流、调研报告、专题发言、工作方案说明。

优先处理有正式受众的材料，例如党委会、总办会、集团汇报、专项工作会、内部交流会。

## 必要输入

| 输入 | 说明 |
| --- | --- |
| 材料类型 | 讲话稿、汇报材料、总结计划、经验交流、调研报告 |
| 使用场合 | 会议名称、汇报对象、内部或外部 |
| 主题 | 本次材料要解决或表达的核心事项 |
| 事实基础 | 工作进展、数据、案例、问题、已采取措施 |
| 时间范围 | 本月、本季度、年度、专项周期 |
| 篇幅 | 提纲、短稿、完整稿 |
| 口径 | 务实、正式、党建融合、问题导向、宣传导向 |

缺失关键事实时，不要编造，用 `[待补充]` 标记。

## 工作步骤

1. 判断材料用途：汇报成绩、说明问题、争取支持、推动落实、交流经验。
2. 搭建结构：背景和要求 -> 工作进展 -> 存在问题 -> 下一步措施 -> 保障机制。
3. 把零散事实归入 3 到 5 个一级标题，避免标题过多。
4. 每个标题下写成“判断 + 事实 + 措施 + 效果或风险”的段落。
5. 对缺少数据的成果表述降级处理，使用“初步形成、持续推进、正在完善”等稳妥表达。
6. 对问题部分写清原因、影响和下一步，不把问题包装成成绩。
7. 输出前检查是否存在空话堆叠、无依据拔高、重复口号。

## 输出结构

### 汇报材料

1. 基本情况
2. 主要工作进展
3. 当前问题和风险
4. 下一步工作安排
5. 需协调事项

### 领导讲话

1. 开场和会议背景
2. 对前期工作的总体评价
3. 对形势任务的判断
4. 对重点工作的部署
5. 对责任落实的要求

### 工作总结

1. 工作开展情况
2. 主要成效
3. 存在不足
4. 改进措施
5. 下一阶段计划

## 禁止事项

- 不编造上级文件、领导要求、会议精神。
- 不编造完成率、金额、排名、覆盖范围。
- 不用“取得历史性突破、形成行业标杆”等高强度判断，除非用户提供证据。
- 不把材料写成营销文案或个人演讲稿。
- 不用空泛排比代替事实。

## 质量自检

- 是否说明了材料用于哪个场景。
- 是否每个成果都有事实或数据支撑。
- 是否有问题和下一步，而不是只有成绩。
- 是否出现责任主体、时间节点、保障机制。
- 是否把缺失事实标成 `[待补充]`。
- 是否能直接提交给办公室或领导初审。

## 示例任务

输入：

> 根据以下要点写一份集团内部汇报：本季度推进数字化台账，覆盖 6 家单位，发现数据口径不一致、责任人更新不及时两个问题，下季度要统一模板、建立月度复盘。

输出应包含：

- 本季度工作背景
- 覆盖范围和已完成动作
- 两个问题的客观表述
- 下季度模板统一、月度复盘、责任更新机制
- `[待补充数据]` 占位，例如具体系统名称、完成率、牵头部门
